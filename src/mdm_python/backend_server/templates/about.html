Die beiden Scraper laufen täglich über einen gitHub-Action-Workflow. Die Daten werden als täglicher Durchschnitt aggregiert aus der Datenbank geladen, um Plots mit Bokeh zu erstellen. Anschliessend wird ein linearer Zusammenhang zwischen Wetter und Energie-Produktion gesucht und die relevanten Wetter-Features werden mit einem ARIMA-Modell modelliert, um eine 10-Tages-Prediction zu erstellen. Anhand dieser Prediction wird versucht die Energie-Produktion für die kommenden 10 Tage zu prognostizieren. Die Ergebnisse werden in einem Front-End mithilfe eines Docker-Containers und mit gitHub-Action-Workflow automatisiert in die Azure-Cloud deployed. Das Back-End dazu wird mithilfe von Flask erstellt und mit Jinja2-Templates gerendert. Der Code ist mithilfe von Dokumentationen, Stack Overflow und ChatGPT eigenständig erstellt worden. {% endblock %} {% block script %}{{ super() }}{% endblock %}
